# task-tracker-pet-project

#Название проекта: Task Tracker

#Цель
Создать инструмент для управления личными и командными задачами с функционалом: создание, изменение статуса, назначение исполнителя, комментарии.
Необходимо для быстроты фиксации задач и прозрачности их жизненного цикла.

#Границы системы
Входит (Scope)
Регистрация / аутентификация пользователя
Создание задачи с заголовком, описанием, дедлайном, исполнителем
Изменение статуса: Открыта → В работе → На проверке → Выполнена / Отклонена
Назначение одного исполнителя на задачу
Добавление комментариев к задаче
Просмотр списка своих задач

Не входит (Out of scope)
Интеграция с календарями (Google, Outlook)
Напоминания по email / push
Групповые задачи (один таск на нескольких)
Детальная аналитика и отчёты
Мобильное приложение (только веб)
Файловые вложения

#Стейкхолдеры
Пользователь
Администратор 

#Требования

Функциональные требования
FR-1 Пользователь может создать задачу с заголовком, описанием (опционально), дедлайном (обязательно, формат YYYY-MM-DD) и исполнителем (если не выбран – назначается на себя).
FR-2 Задача имеет статус: Открыта → В работе → На проверке → Выполнена. Админ может отклонить с комментарием (вернуть в В работе).
FR-3 Пользователь видит только задачи, где он создатель или исполнитель. Админ видит все.
FR-4 Пользователь может добавить комментарий к любой видимой задаче.
FR-5 Система должна работать без подвисаний при списке до 1000 задач (просто упоминание, без цифр SLA).

Нефункциональные требования
NFR-1 Простота использования-создать новую задачу можно за 2 клика: кнопка «+» -> заполнить заголовок и дедлайн -> сохранить.
NFR-2 Производительность-список задач (до 100 штук) должен отображаться мгновенно, без задержек (меньше 1 секунды). 
NFR-3 Безопасность-пользователь не должен видеть чужие задачи через прямой URL (реализуется проверкой на бэкенде). 
NFR-4 Доступность-веб-интерфейс должен работать в последних версиях Chrome, Firefox и Edge (адаптация не нужна, достаточно десктопной версии).
NFR-5 Надёжность-при попытке отправить форму с пустым дедлайном система не должна падать, а показывать понятную ошибку прямо на форме (без перезагрузки страницы).
